Este catálogo se conecta a internet (endoflife.date + osv.dev). Si estás detrás de un proxy corporativo, indícalo aquí. Usa el almacén de certificados del sistema operativo, así que las CA de inspección (Zscaler, etc.) ya confiadas por Windows funcionan automáticamente.

// QA · auditar manifiesto de dependencias

Pega o sube el package.json (npm/web) o el build.gradle.kts / settings.gradle.kts (Android/Gradle) de un desarrollo. Por cada dependencia verás: versión declarada, última disponible (npm o Maven Central / Google Maven), cuánto está desfasada (major/minor/patch), CVEs conocidos y el análisis de cumplimiento legal. Las dependencias Gradle se incorporan además al inventario.

⚙ configurar IA (Claude) — opcional

El reporte siempre se genera con el modelo de riesgo local. Si configuras una API key de Anthropic, además se redacta el Análisis y la Conclusión con Claude. La key se guarda localmente (config.json) y se usa a través de tu proxy/CA. Necesita acceso a api.anthropic.com.

// análisis estático de código fuente

Sube el comprimido (.zip, .7z o .rar) del código fuente de la aplicación. Se analiza en un entorno aislado (nunca se ejecuta el código) buscando incumplimientos respecto de la Guía Técnica de Lineamientos para el Desarrollo de Software (División de Gobierno Digital, SEGPRES) y el marco legal chileno (Leyes 19.628, 21.096, 21.180 y 21.459). La evidencia de secretos se muestra redactada (principio de mínima exposición).
